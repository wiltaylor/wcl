# JavaScript / TypeScript Binding (`wcl_wasm`)

WASM bindings generated by `wasm-bindgen`. Source: `bindings/wasm/pkg/wcl_wasm.d.ts`, `bindings/wasm/src/`.

## Install

```bash
npm install wcl_wasm
```

ESM-only (`"type": "module"` in `package.json`). Works in Node and browsers.

## Minimal Example (Node / bundler)

```js
import init, { parse, parseValues, query } from "wcl_wasm";

await init();   // initialise the WASM module once

const src = `
server web {
    host = "0.0.0.0"
    port = 8080
}
`;

const doc = parse(src);
if (doc.hasErrors) {
    for (const d of doc.diagnostics) console.error(d);
} else {
    console.log(doc.values);
}

// Or fetch just values (throws on parse errors)
const values = parseValues(src);

// Or run a query in one call
const ports = query(src, "server | .port");
```

## Core API

From `wcl_wasm.d.ts`:

| Function | Signature | Behavior |
|----------|-----------|----------|
| `parse(source, options?)` | `(string, any?) → { values, hasErrors, diagnostics }` | Full document |
| `parseValues(source, options?)` | `(string, any?) → any` | Just evaluated values; **throws** on parse errors |
| `query(source, queryStr, options?)` | `(string, string, any?) → any` | Parse + run a query; returns the query result |
| `default export __wbg_init` | `(input?) → Promise<InitOutput>` | Initializer — must be awaited before first use |
| `initSync(module)` | `(module) → InitOutput` | Synchronous initializer (when WASM bytes are already in-memory) |

Note: this binding intentionally exposes a flatter API than the others — there is no `Document` class or `blocks_of_type`. If you need block-level access, call `parse` and drill into `values`, or call `query(source, "<kind>")` to get a filtered list.

## Options

Pass as a plain JS object (camelCase keys):

```js
parse(src, {
    rootDir: "/absolute/path",
    allowImports: true,
    maxImportDepth: 32,
    maxMacroDepth: 64,
    maxLoopDepth: 8,
    maxIterations: 10000,
    variables: { PORT: 8080 },
    libPaths: ["/custom/lib"],
    noDefaultLibPaths: false,
});
```

## Value Type Mapping

| WCL | JS/TS |
|-----|-------|
| string | `string` |
| int (≤ 2^53) | `number` |
| int (> 2^53) | `bigint` (JSON-string round-trip loses precision — see gotchas) |
| float | `number` |
| bool | `boolean` |
| null | `null` |
| list | `Array` |
| map | plain object |
| block | plain object with `kind`, `id`, `attributes`, `children`, `decorators` |
| date | ISO string |
| duration | ISO string |
| symbol | string (prefixed `:`) |

## TypeScript

The `.d.ts` exports `any` for values. If you want typed output, declare your own interfaces and cast:

```ts
interface Config {
    server: { host: string; port: number };
}

const cfg = parseValues(src) as Config;
```

## Error Handling

- `parse` returns `{ hasErrors, diagnostics }` — never throws for WCL errors.
- `parseValues` **throws** an `Error` when `hasErrors` is true (with concatenated messages).
- `query` throws when the query string itself is invalid.

## Gotchas

- **`await init()` before first call** — the WASM module needs to be instantiated.
- **`bigint` for large ints** — precision loss for integers outside `Number.MAX_SAFE_INTEGER`. If you need `i64` or `u64`, parse into a `bigint` downstream.
- **Custom functions are not currently exposed in this binding** — use one of the other bindings (Go / Python / .NET / JVM / Rust) if you need `functions`.
- **File I/O** — the WASM module cannot access the host filesystem. To parse a file, read it in JS and pass the string; set `rootDir` manually if you use relative imports.
- Import attribute support in bundlers varies; if the `.wasm` file isn't served, use `initSync` with the bytes you fetched.
